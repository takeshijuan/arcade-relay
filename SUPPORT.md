# Support

Use the right channel so maintainers can respond efficiently.

## Questions

Open a GitHub issue using the question template, or use GitHub Discussions if
the repository enables them.

Include:

- What you are trying to build.
- Which command you ran, such as `/forge` or `/forge-status`.
- Relevant `state/stage.txt` and `state/active.md` details.
- Whether provider keys are configured or local degraded mode is being used.

## Bugs

Use the bug report issue template. Include logs, screenshots, and the generated
`qa/report.md` when the failure involves a playable game.

## Security

Follow [SECURITY.md](SECURITY.md). Do not put vulnerability details or secrets in
public issues.

## Feature Requests

Use the feature request template and describe the target workflow, phase, or
agent boundary.
