## DR-CONCEPT iteration 1 — CONCERNS
- 日時: 2026-07-10T14:30:00+09:00
- 指摘要約: （優先度順）
  1. 【要修正・GDD前提】自動攻撃の性質が文書内で矛盾。一文コンセプト/コアループは「群れを薙ぎ払う」(AoE殲滅) だが Mechanics/P-02 は「最寄り敵へ周期発動」(単体寄り)。AoEか単体かで P-02 の無双感と P-01/P-03 の「囲まれる」ダイナミクスの成否が両方変わる。攻撃範囲・形状（ヒーロー中心の放射AoE / 最寄り単体 のどちらか）を明記し、一文・コアループ・Mechanics・P-02 で一貫させること。
  2. 【要修正・面白さ仮説の成立前提】カメラ方式が全文で未定義。3Dで「四方から迫る群れ」を知覚できるかはカメラ依存で、俯瞰固定か追従見下ろしかで P-01（見えない敵は躱せない）・P-03・「1画面で成立」が変わる。コアループにカメラ方式を1文で明記し、四方の敵を同時視認できることを担保すること。
  3. 【設計リスクの明示】P-02 強度と P-01/P-03 の中心的緊張が未言明。自動攻撃が群れを掃討しきると P-01 の「囲まれる状況」が発生せず、弱いと P-02 の無双感が消える。この均衡が面白さの成否を握るので「検証したい面白さの仮説」かスコープに最大リスクとして明記し、GDDで解像度を上げる方針（例: 自動攻撃は殲滅でなく間引き＝密度が必ず上回る）を1行添えること。
  4. 【GDD申し送り可】恒久アップグレードのパワークリープと「2〜3分セッション/エンドレス難度」の整合が未記述。恒久強化で序盤が作業化、またはランが長期化して想定セッション長と乖離するリスク。難度カーブが恒久強化を吸収する方針をGDDで1行示すこと（concept段階では申し送りで可）。
- 対応:
  1. 対応済み。攻撃を「最寄り1体への単体ヒット（AoEではない）」に統一し、一文コンセプト・コアループ・MDA Mechanics・新設「設計判断」節の全箇所で表記を揃えた。「薙ぎ払う」という語も一文コンセプトから外し「間引く」に置換して AoE を連想させる表現を除去した。
  2. 対応済み。カメラ方式を「固定俯瞰（見下ろし角60〜70度、アリーナ全体を常時画角に収める）」として一文コンセプト・コアループ・MDA Mechanics・設計判断節に明記し、四方の敵を同時視認できることを担保する設計とした。
  3. 対応済み。新設「設計判断」節に P-02 と P-01/P-03 の均衡を最大設計リスクとして明記し、解像度を上げる方針（自動攻撃は間引き型＝単体ヒット・DPS上限を群れの増加ペースが常に上回るよう難度カーブと連動）を1行で示した。詳細な初期値・調整レンジは GDD 側で規定する申し送りとした。
  4. 対応済み（GDD申し送り）。「設計判断」節に恒久アップグレードのパワークリープとセッション長（2〜3分）の整合方針（難度カーブをベース強度に対する相対値で定義し恒久強化幅を吸収する）を1行で明記した。具体的な数値は GDD で初期値＋調整レンジとして規定する。

## DR-CONCEPT iteration 2 — CONCERNS
- 日時: 2026-07-10T15:10:00+09:00
- 指摘要約: （優先度順）
  1. 【要修正・反証可能性】最大設計リスクと自認した P-02（照準ゼロ自動攻撃＝無双感）の成否を反証する仮説が「検証したい面白さの仮説」に欠落。既存2仮説は P-01（紙一重回避）と P-04（再挑戦）のみをカバーし、iteration1 の設計変更（AoE不採用・単体ヒット・「薙ぎ払う」削除・DPS上限）が brief Core Fantasy「自動攻撃で薙ぎ払う無双感」/ MDA「操作せず薙ぎ払う無双感（→P-02）」を構造的に生めるか（＝チクチク削りに感じないか）が Checkpoint B で反証できる形になっていない。合格条件: (a) 無双感の成立を観察可能な行動/実感で反証できる仮説を1つ追加、または (b) P-02 の Aesthetic 表記を単体ヒットの実態（群れを捌く/いなす快感 等）へ整合し brief Core Fantasy との差分を「設計判断」に1行明記。
  2. 【品質向上】敵種数「1〜2種」が範囲指定のまま。P-03（敵種多様化せず物量で難度・敵3種目以降却下）に照らせば基本1種で成立するのが本筋だが、最小成立ラインと2種目の位置づけ（必須/任意）が未記載で GDD がぶれうる。合格条件: 「基本1種で成立、2種目は任意の見た目/挙動差分」等、下限と2種目の扱いを1行明記。
  3. 【GDD申し送り】単体ヒット自動攻撃の当たり表現（近接スイング/飛び道具/ヒットエフェクト）が未定義。brief 確保済みの hero attack アニメクリップ・攻撃ヒット SFX と資産計画を整合させるため、方式を GDD で確定する申し送りを1行残すこと（concept 段階の確定は不要）。
- 対応:
  1. 対応済み（(a)(b) 両方で対応）。MDA の P-02 Aesthetic 表記を「操作不要で敵を確実に仕留め続け群れを捌いていく手応え」に調整し brief Core Fantasy（薙ぎ払う無双感）との差分を「設計判断」節に新規1行として明記した。加えて「検証したい面白さの仮説」に、単体ヒット自動攻撃が単調な作業感に落ちず無双寄りの手応えとして成立するかを反証可能な形で問う仮説を1件追加した（計3仮説、P-01/P-02/P-04をカバー）。
  2. 対応済み。スコープ「In scope」の敵記述を「基本1種で成立を完結させ、2種目は行動ロジックを増やさない見た目/軽微なステータス差分の任意追加（必須ではない）」に書き換え、P-03（物量・速度で難度を作り敵種を増やさない）との整合を明記した。「設計判断」節にも下限と2種目の位置づけを1行で追記した。
  3. 対応済み（GDD申し送り）。「設計判断」節に、当たり表現（近接スイング/飛び道具/瞬間ヒット）の確定は GDD 側の責務とし、brief 確保済みの hero attack アニメクリップ・攻撃ヒット SFX と矛盾しない方式を選ぶ旨を1行追記した。
