# design-docs — design/ 編集時の強制規約

design/ の文書は下流（実装・資産生成・QA）が機械的に参照する。ID とピラーの規律は contract.md §8 が正本。

## Do / Don't

- **Do**: 全システム・全資産エントリは `design/concept.md` のピラー `P-xx` を明示参照する（`pillar: P-01` / 「(P-02)」等）。gdd.md のシステム、assets.md の各資産に必須
- **Don't**: どのピラーにも寄与しない（＝参照を書けない）システム/資産を追加しない。書けないなら削るかピラーを見直す
- **Do**: ゲームパラメータは**初期値＋調整レンジ**で書く（例: `ジャンプ力: 420 px/s（調整レンジ 360–500）`）。実装側は初期値を config.ts に写す
- **Don't**: 「後で決める」「適宜調整」「速め」等の未確定値・形容詞のみの数値指定を書かない
- **Do**: 安定ID（ピラー `P-xx`、ストーリー `S-xx`）は一度振ったら不変。廃止は削除ではなく status/注記（`（廃止: 理由）`）で示す
- **Don't**: ID の振り直し・欠番詰め・再利用をしない（下流の参照が全部壊れる）
- **Do**: 各文書は `.claude/docs/templates/` の対応テンプレの見出し構成に従う。テンプレの節を勝手に削除・改名しない（追加節はテンプレ節の後ろに置く）
- **Do**: ピラーは3〜5個、互いに独立で、意思決定の裁定に使える具体性を持たせる（DR-CONCEPT ゲート基準）

## 正誤例

### ピラー参照

```markdown
<!-- NG: どのピラーに効くのか不明 -->
## システム: コンボ
連続ヒットで倍率が上がる。

<!-- OK -->
## システム: コンボ（P-02: 途切れない爽快感）
連続ヒットで倍率が上がる。P-02 のため、コンボ猶予は操作ミス1回で切れない長さにする。
```

### 数値は初期値＋調整レンジ

```markdown
<!-- NG -->
プレイヤー速度: 速め。敵HP: 後で決める。

<!-- OK -->
| パラメータ | 初期値 | 調整レンジ | 根拠 |
|---|---|---|---|
| プレイヤー移動速度 | 220 px/s | 180–280 | P-02: 1画面を約3秒で横断 |
| 敵HP（雑魚） | 3 | 2–5 | 3ヒットで撃破の手応え |
```

### 安定ID

```markdown
<!-- NG: P-03 を削除して P-04 を P-03 に繰り上げ -->
- P-01: ...
- P-02: ...
- P-03: （旧P-04の内容）

<!-- OK: 廃止を注記で示し、番号は不変 -->
- P-01: ...
- P-02: ...
- P-03: （廃止: スコープ過大のため Checkpoint A で削除決定）
- P-04: ...
```
